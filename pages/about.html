Fetch Connection

In a modern web application architecture, the connection between the backend and frontend is crucial for seamless functionality. When utilizing Amazon Web Services (AWS) to deploy the backend and GitHub Pages to host the frontend, this connection is established through a well-defined pipeline. AWS offers a robust infrastructure for hosting and managing the backend services, ensuring scalability, reliability, and security. On the other hand, GitHub Pages simplifies the deployment of the frontend by providing an easy-to-use platform for serving static content. The backend, hosted on AWS, communicates with the frontend hosted on GitHub Pages through APIs and web requests, creating a bridge that allows data transfer, user interactions, and a cohesive user experience. This setup not only streamlines development and deployment processes but also provides a cost-effective and reliable solution for web applications.

Jekyll with HTML

LINK

Click the link above for layout file of this "about page" to learn more about Jekyll with HTML.

The layout file begins with a section of YAML front matter enclosed by triple dashes (---). In Jekyll, front matter is used to define metadata for the page. In this case, it specifies the layout to use ("default") and the title of the page ("About").

Linking CSS and JavaScript: You include external CSS and JavaScript files using the HTML link and script elements. In this case, you are referencing CSS and JavaScript files from the "assets/css" and "assets/scripts" directories, respectively. Note the use of {{ site.baseurl }} in the file paths. This allows Jekyll to generate the correct paths when building the site, taking into account the base URL of your site.
